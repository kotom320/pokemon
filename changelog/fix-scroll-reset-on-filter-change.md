# 필터 변경 시 스크롤 초기화 및 과다 호출 수정

날짜: 2026-04-07

## 버그 수정

- **필터 변경 시 스크롤이 이전 위치에 남아있는 문제**
  - 증상: 스크롤을 내린 상태에서 세대/타입 필터를 바꾸면 스크롤이 이전 위치에 그대로 남음
  - 수정: 마운트 시 `scrollMap`에 저장된 값이 없으면 `window.scrollTo(0, 0)` 호출

- **필터 변경 후 API 과다 호출 문제**
  - 증상: 스크롤을 많이 내린 뒤 필터를 변경하면 sentinel이 즉시 viewport에 들어와 연속 fetchNextPage 발생
  - 원인: 필터 변경 시 컴포넌트가 리마운트되지만 window 스크롤은 이전 위치 유지 → sentinel이 바로 보임
  - 수정: 위의 스크롤 초기화로 함께 해결 (top으로 이동하면 sentinel이 viewport 밖으로 나감)
